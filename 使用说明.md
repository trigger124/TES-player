# TES 比赛实时 AI 监听系统 - 配置与运行指南

> 下载本项目后，按下面步骤操作即可运行。

---

## 一、环境准备

### 1. 硬件要求

- Windows 10 / Windows 11
- NVIDIA 独立显卡（推荐 RTX 3060 及以上）
- 显存 8GB 及以上（Whisper medium 模型约占用 3GB）
- 能正常播放声音的扬声器或耳机

### 2. 软件要求

- Python 3.12（**不要用 Anaconda，建议用官方 Python**）
- NVIDIA 显卡驱动（建议 560 系列或更高，支持 CUDA 12.6）
- 一个可用的 DeepSeek API Key

---

## 二、安装 Python 3.12

1. 访问 [Python 官网下载页](https://www.python.org/downloads/release/python-3124/)
2. 下载 `Windows installer (64-bit)`，文件名类似 `python-3.12.4-amd64.exe`
3. 安装时务必勾选：
   - ✅ **Add python.exe to PATH**
   - ✅ **Use admin privileges when installing py.exe**
4. 建议安装路径保持默认：`C:\Program Files\Python312`

安装完成后，打开 PowerShell 验证：

```powershell
python --version
```

应输出：`Python 3.12.4`

---

## 三、下载项目代码

```powershell
git clone https://github.com/你的用户名/tes-ai-monitor.git
cd tes-ai-monitor
```

或者直接在 GitHub 页面点击 `Code → Download ZIP`，解压到任意目录。

## 四、创建虚拟环境

在项目目录下打开 PowerShell，执行：

```powershell
python -m venv venv
```

激活虚拟环境：

```powershell
venv\Scripts\activate
```

激活成功后，命令行前面会出现 `(venv)` 字样。

---

## 五、安装依赖

### 1. 安装 PyTorch（CUDA 12.6 版本）

#### 方式一：官方 pip 安装（推荐网络好时使用）

```powershell
pip install torch==2.12.1+cu126 torchaudio==2.11.0+cu126 --index-url https://download.pytorch.org/whl/cu126
```

#### 方式二：清华镜像安装

如果官方下载慢或超时，可以使用清华镜像：

```powershell
pip install torch==2.12.1+cu126 torchaudio==2.11.0+cu126 --index-url https://mirrors.tuna.tsinghua.edu.cn/pytorch-wheels/cu126
```

#### 方式三：本地 wheel 安装

如果你网络环境无法直接从 pip 下载，可以先通过网盘下载本地 wheel 文件，再安装。

**第一步：下载 wheel 文件**

请从以下网盘链接下载 `torch-2.12.1+cu126-cp312-cp312-win_amd64.whl`：

```text
[你的网盘链接]
提取码 / 访问密码：[你的密码]
```

下载完成后，把 wheel 文件放到项目根目录（和 `main_tes.py` 同级）。

**第二步：本地安装**

```powershell
pip install "torch-2.12.1+cu126-cp312-cp312-win_amd64.whl"
pip install torchaudio==2.11.0+cu126 --index-url https://download.pytorch.org/whl/cu126
```

> **注意**：这个 wheel 文件约 2.8GB，无法直接上传到 GitHub 免费仓库（普通文件 100MB 限制，LFS 2GB 限制），因此请通过网盘或对象存储分享下载链接。

### 2. 安装其他 Python 包

```powershell
pip install openai-whisper silero-vad pyaudiowpatch openai numpy==1.26.4
```

> `numpy==1.26.4` 必须指定版本，numpy 2.x 与当前 torch 不兼容。

## 六、配置 DeepSeek API Key

项目中的 `config.example.py` 是配置模板，你需要把它复制成 `config.py`，然后填入自己的 API Key。

### 1. 复制配置文件

```powershell
copy config.example.py config.py
```

### 2. 获取 DeepSeek API Key

1. 访问 [DeepSeek 开放平台](https://platform.deepseek.com/)
2. 注册并登录账号
3. 进入 "API Keys" 页面，点击 "创建 API Key"
4. 复制生成的 Key（格式类似 `sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`）

### 3. 修改 config.py

用文本编辑器打开 `config.py`，找到这一行：

```python
'api_key': 'your-deepseek-api-key-here',
```

替换成你刚才复制的真实 Key：

```python
'api_key': 'sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx',
```

其他配置项说明：

| 配置项                   | 说明                     | 默认值                          |
| ------------------------ | ------------------------ | ------------------------------- |
| `api_key`              | DeepSeek API Key         | 必填                            |
| `api_base`             | DeepSeek API 地址        | `https://api.deepseek.com/v1` |
| `model`                | 使用的模型               | `deepseek-chat`               |
| `context_window_size`  | 上下文窗口大小           | `5`                           |
| `comfort_video_url`    | 失败时打开的安慰视频链接 | B 站《突然的陀螺》              |
| `confidence_threshold` | 低置信度复核阈值         | `0.85`                        |
| `victory_keywords`     | 胜利关键词列表           | 见 config.py                    |
| `defeat_keywords`      | 失败关键词列表           | 见 config.py                    |

## 七、运行项目

确保虚拟环境已激活（命令行前有 `(venv)`），然后执行：

```powershell
python main_tes.py
```

首次运行会自动下载 Whisper medium 模型（约 1.5GB），请保持网络畅通。

看到类似输出说明启动成功：

```text
Whisper device auto: cuda
Silero VAD 模型加载成功
使用设备: 扬声器 (Realtek(R) Audio) [Loopback]
[实时状态] 初始状态 | 监听中...
```

## 八、使用演示

1. 打开 B 站、虎牙等视频平台
2. 播放任意 TES 比赛直播或回放
3. 保持系统音量正常输出
4. 观察终端实时输出：
   - `[检测到语音]` 表示 VAD 检测到人声
   - `[Whisper识别]` 显示识别出的解说文字
   - `[实时状态]` 显示 DeepSeek 推断的比赛状态
5. 当 AI 判定 TES 失败时，会自动打开安慰视频
6. 当 AI 判定 TES 胜利时，会自动结束监听

## 九、常见问题

### 1. 启动时报 `OSError: [WinError 1114]`

**原因**：VC++ 运行时版本不匹配，通常是因为用了 Anaconda Python。
**解决**：卸载 Anaconda Python，安装官方 Python 3.12，并重新创建虚拟环境。

### 2. `torch.cuda.is_available()` 返回 False

**原因**：显卡驱动不支持 CUDA 12.6，或安装了 CPU 版 PyTorch。
**解决**：更新 NVIDIA 驱动，并重新安装 CUDA 12.6 版本的 PyTorch。

### 3. Silero VAD 报 `Input audio chunk is too short`

**原因**：音频 chunk 重采样后不足 512 样本。
**解决**：检查是否修改过 `audio_capture.py` 中的 `chunk_size`，保持默认 1024 即可。

### 4. 检测不到解说声音

**原因**：可能没捕获到扬声器回环，或 PyCharm/IDE 解释器选错。**解决**：

- 确认系统有声音输出
- 检查终端第一行 `使用设备:` 是不是 `[Loopback]` 设备
- 确保运行用的是项目虚拟环境 `venv`，不是系统 Python 或其他环境

### 5. 模型下载很慢或失败

**原因**：Whisper 模型需要从 HuggingFace/OpenAI 服务器下载。
**解决**：开启科学上网工具，或手动下载 `medium.pt` 放到 `C:\Users\你的用户名\.cache\whisper\` 目录。
